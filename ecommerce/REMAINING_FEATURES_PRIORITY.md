# 功能缺口优先级清单

> 目标：把当前电商小程序补成“可真实交易、可交付、可上线”的版本。

## P0：必须优先完成

### 1. 真实支付接入
- [ ] 申请微信支付商户号
- [ ] 配置商户 API 密钥 / 证书
- [ ] 实现微信统一下单
- [ ] 实现支付回调验签
- [ ] 支付成功后更新订单状态为 `paid`
- [ ] 支付失败重试/提示

**说明：**
这是电商最核心的一步，没有真实支付就无法完成闭环。

### 2. 支付成功通知
- [ ] 开通小程序订阅消息能力
- [ ] 配置订阅消息模板 ID
- [ ] 结算页请求用户订阅授权
- [ ] 支付成功后发送购买清单消息
- [ ] 消息中包含订单号、商品、金额、状态

**说明：**
你希望“买完后发条信息”，这部分必须补齐。

### 3. 订单状态闭环
- [ ] 统一订单状态定义
- [ ] 下单后状态为 `pending`
- [ ] 支付成功后状态为 `paid`
- [ ] 商家发货后状态为 `shipped`
- [ ] 用户确认收货后状态为 `completed`
- [ ] 取消订单后状态为 `cancelled`

**说明：**
状态流转要和支付、发货、售后完全对应。
