# 微信小程序发布指南 - 从开发到上线

## 📋 发布前需要完成的工作

### 1️⃣ 开发者账号准备

#### **个人账号 vs 企业账号**

| 账号类型 | 商业功能 | 支付权限 | 审核难度 | 建议 |
|---------|---------|---------|---------|------|
| 个人 | ❌ 不支持 | ❌ 无法接入微信支付 | ⭐ 容易 | ❌ 不适合电商 |
| 个体工商户 | ⚠️ 有限制 | ⚠️ 需特殊申请 | ⭐⭐ 中等 | ⚠️ 可用但复杂 |
| **企业** | ✅ 完全支持 | ✅ 完全支持 | ⭐⭐⭐ 需审核 | **✅ 推荐** |
| 政府/媒体 | ✅ 完全支持 | ✅ 完全支持 | ⭐⭐ 相对容易 | ✅ 也可以 |

**结论**：做电商小程序，你**必须用企业账号**（或个体工商户）

---

### 2️⃣ 企业资质准备

需要准备以下材料：

```
📦 企业基本信息
├─ 营业执照 (扫描件 / 照片)
├─ 法定代表人身份证 (正反面)
├─ 企业联系方式
└─ 企业简介

📦 小程序账号信息
├─ 小程序名称 (需要审核，不可违规)
├─ 小程序描述 (简洁说明功能)
├─ 小程序分类 (电商为主)
├─ 小程序logo (1024x1024px)
└─ 运营主体 (必须是你的企业)

📦 支付权限申请 (可选，但电商必需)
├─ 结算账户 (企业对公账户或法人个人账户)
├─ 结算周期 (通常 T+1)
└─ 结算手续费 (一般 0.6%)
```

## 📱 发布步骤详解

### 步骤 1：注册微信小程序账号

**地址**：https://mp.weixin.qq.com/

1. 点击"立即注册"
2. 选择"小程序"
3. 邮箱验证
4. 选择**企业类型**认证
5. 上传营业执照和法人身份证
6. 等待审核（1-3 天）

![小程序注册流程]

---

### 步骤 2：完成小程序基本设置

登录 **https://mp.weixin.qq.com/wxamp/home**

#### 2.1 基本信息设置
```
设置 → 基本信息

小程序名称：
  ├─ 精准描述业务（如"XX商城"）
  ├─ 包含关键词（便于搜索发现）
  └─ 不超过30个汉字

小程序描述：
  └─ 简洁说明核心功能，不超过120字

服务类目：
  └─ 选择【电商】→【在线商城】

小程序logo：
  ├─ 1024 × 1024px
  ├─ PNG或JPG格式
  ├─ 清晰识别度高
  └─ 不包含个人信息
```

#### 2.2 隐私政策和用户协议
```
设置 → 基本信息 → 隐私保护

实名认证：【已完成】

隐私政策：
  ├─ 新增文档
  ├─ 选择"电商类"模板
  ├─ 编写你的隐私政策
  └─ 提交审核

用户协议：
  ├─ 同上
  └─ 新增用户协议
```

**隐私政策关键内容**：
```markdown
# 隐私政策

## 1. 我们收集什么信息

- 用户昵称、头像（微信授权）
- 收货地址、手机号（订单必需）
- 浏览历史、购物数据（服务优化）

## 2. 信息如何使用

- 用于订单处理和物流通知
- 用于优化推荐和服务
- 不用于任何商业售卖

## 3. 信息如何保护

- 使用 HTTPS 加密传输
- 后端数据库加密存储
- 定期安全审计

## 4. 用户权利

- 用户可申请导出个人数据
- 用户可申请删除个人信息
- 用户可撤销授权

## 5. 联系我们

- 隐私问题反馈：xxx@xxx.com
- 投诉处理流程：7 个工作日内回复
```

### 步骤 3：配置服务器域名

```
设置 → 开发设置 → 服务器域名

request 合法域名：
  └─ https://your-api.com/api
  
socket 合法域名：
  └─ （如有即时通讯，配置 WebSocket）

uploadFile 合法域名：
  └─ https://your-upload.com/upload

downloadFile 合法域名：
  └─ https://your-resource.com/files
```

**⚠️ 重要**：
- 必须是 **HTTPS** 协议
- 必须通过企业备案的域名
- 支持国企/政府/私企域名
- 不支持只有 IP 地址

### 步骤 4：申请微信支付权限

需要满足条件：
- ✅ 企业账号认证已完成
- ✅ 小程序已发布上线

#### 4.1 进入支付配置

```
微信小程序后台
  → 功能 → 微信支付
  → 前往支付申请
```

#### 4.2 填写结算信息

```
商户号申请表

商户名称：你的企业名
商户简介：电商平台简介
结算账户：企业对公账户或法人个账
行业代码：5411（超市里的百货店）或 5814（电商平台）
月交易额预估：自填（会影响额度）
```

#### 4.3 签约和费率

```
费率体系（一般）：
  ├─ 基础费率：0.60%
  ├─ 支付成功后自动扣款
  └─ T+1 结算到账户
```

### 步骤 7：提交审核

#### 7.1 进入管理后台

```
https://mp.weixin.qq.com/wxamp/versions
  → 选择最新版本
  → 点击"提交审核"
```

#### 7.2 填写审核信息

```
业务功能：
  ├─ 商品浏览
  ├─ 购物车功能
  ├─ 订单创建
  ├─ 在线支付
  └─ 订单追踪

测试账号：
  ├─ 账号：test001 (可选，建议填)
  ├─ 密码：test123456
  └─ 备注：用于审核人员测试

备注说明：
  └─ "这是一个在线商城小程序，支持商品浏览、
     购物车、订单创建、微信支付等功能。
     测试账号可直接登录体验。"

使用场景：
  └─ 电商购物平台

隐私政策链接：
  └─ https://your-domain.com/privacy-policy
```

#### 7.3 常见审核驳回原因及解决

| 驳回原因 | 解决办法 |
|---------|---------|
| 隐私政策不完整 | 补充完整的隐私政策，包含数据收集、使用、保护说明 |
| 功能描述与实现不符 | 确保 app.json 的描述与实际功能一致 |
| 包含诱导分享 | 移除任何要求分享才能获益的功能 |
| 页面不稳定 | 测试所有页面，修复 bug，确保流畅运行 |
| 商品信息虚假 | 确保商品、价格、描述真实有效 |
| 支付功能异常 | 测试支付流程，确保正常完成 |
| 存在违规词汇 | 搜索并替换所有违禁词汇 |

**驳回后**：修复 → 重新上传 → 重新提交审核

## 🔄 发布后的维护

### 版本更新流程

```
有更新需求
  ↓
修改代码 (本地测试通过)
  ↓
上传新版本 (开发工具)
  ↓
提交审核
  ↓
等待审核 (1-3 天)
  ↓
审核通过
  ↓
发布 (用户自动收到更新提示)
```

### 灰度发布（推荐）

```
假设当前是 v1.0，要发布 v2.0

后台设置 → 灰度发布

选择 v2.0 版本
  ├─ 灰度比例：20% (先给 20% 用户)
  ├─ 观察 24h 数据
  ├─ 无问题 → 提升到 50%
  ├─ 继续观察 24h
  └─ 最后 → 100% 全量发布

优点：降低风险，出问题只影响部分用户
```

## ⚙️ 常见问题

### Q1: 我是个人开发者可以做电商吗？
**A**: ❌ 不可以。微信明确规定电商类功能只有企业账号可用。需要注册企业小程序账号。

### Q2: 支付权限申请需要多久？
**A**: 通常 5-10 个工作日。建议：
- 资料准备充分
- 及时回复客服问题
- 在工作时间跟进

### Q3: 审核驳回了怎么办？
**A**: 
1. 仔细阅读驳回原因
2. 本地修复问题
3. 重新上传版本
4. 重新提交审核
5. 通常第二次通过率 > 90%

### Q4: 上线后发现 bug 怎么办？
**A**: 
- 紧急 bug：下线版本 → 修复 → 灰度 5% 测试 → 重新发布
- 普通 bug：正常更新流程

### Q5: 用户投诉导致下架了怎么办？
**A**:
1. 分析投诉原因
2. 进行产品调整
3. 向平台申请恢复
4. 通常可在整改后恢复
